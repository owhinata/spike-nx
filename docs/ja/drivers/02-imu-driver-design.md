# IMU ドライバ設計

## 1. 概要

B-L4S5I-IOT01A Discovery ボード上の LSM6DSL（6 軸: 加速度+ジャイロ）を開発ターゲットとして採用。最終目標は pybricks の LSM6DS3TR-C ドライバと同等の機能を NuttX 上で実現すること。

LSM6DSL と LSM6DS3TR-C（SPIKE Prime Hub 搭載）は同じ ST IMU ファミリーで、レジスタ構成に互換性がある。IMU 処理ライブラリ (apps/imu/) はセンサー非依存のため、センサー間の移行時にも変更不要。

---

## 2. デバイス: LSM6DSL

### アーキテクチャ

LSM6DSL は 3 軸加速度計と 3 軸ジャイロスコープを統合したシングルダイ 6 軸 IMU。

| センサー | I2C アドレス (SDO/SA0) | WHO_AM_I |
|----------|------------------------|----------|
| 加速度 + ジャイロ | 0x6A (LOW) / 0x6B (HIGH) | 0x6A |

### 性能仕様

| パラメータ | ジャイロ | 加速度 |
|-----------|---------|--------|
| 最大 ODR | 6.66 kHz | 6.66 kHz |
| フルスケール | 125 / 250 / 500 / 1000 / 2000 dps | ±2 / 4 / 8 / 16 g |

### データ取得（pybricks パターン）

INT1 (gyro DRDY) 1 本の割り込みで 12 バイト一括読み出し:
- バイト 0-5: ジャイロ X/Y/Z (OUTX_L_G 0x22 〜 OUTZ_H_G 0x27)
- バイト 6-11: 加速度 X/Y/Z (OUTX_L_A 0x28 〜 OUTZ_H_A 0x2D)

両センサーが同一 ODR で動作するため、gyro DRDY 時に accel データも更新済み。

### 主要レジスタ設定

| レジスタ | 設定 | 目的 |
|----------|------|------|
| CTRL3_C (0x12) | BDU=1, IF_INC=1 | データ更新ブロック、アドレス自動インクリメント |
| CTRL5_C (0x14) | ROUNDING=011 | バースト読み出し用ラウンディング |
| DRDY_PULSE_CFG (0x0B) | DRDY_PULSED=1 | パルスモード DRDY（より確実） |
| INT1_CTRL (0x0D) | INT1_DRDY_G=1 | ジャイロ DRDY を INT1 にルーティング |

## 3. NuttX ドライバアーキテクチャ

ドライバは nuttx fork に uORB センサードライバとして実装。

```
[カーネル空間]
  lsm6dsl_uorb.h    - config 構造体、登録 API
  lsm6dsl_uorb.c    - I2C ヘルパー、バースト読み出し、uORB 登録

[uORB トピック]
  /dev/uorb/sensor_accel0   (m/s²)
  /dev/uorb/sensor_gyro0    (rad/s)
```

### データ取得モード

| モード | メカニズム | 使用条件 |
|--------|-----------|----------|
| 割り込み | INT1 DRDY → HPWORK → バースト読み出し | ボードが `attach` コールバックを提供 |
| ポーリング | kthread スリープ → バースト読み出し | `attach = NULL` |

### デフォルトスケール設定

| センサー | デフォルト | 理由 |
|----------|-----------|------|
| 加速度 | ±2 g | チップリセットデフォルト、姿勢検出に十分 |
| ジャイロ | 250 dps | チップリセットデフォルト |

---

## 4. データフロー

```
INT1 (gyro DRDY) 発火
  → HPWORK: 0x22 から 12 バイト一括読み出し
  → raw int16 × 6 (ジャイロ XYZ + 加速度 XYZ)
  → float スケール乗算
  → push_event で sensor_gyro0 と sensor_accel0 に配信
```

加速度とジャイロのイベントは同一タイムスタンプを共有し、同期データを保証。

## 5. IMU 処理ライブラリ (apps/imu/)

uORB データを消費するセンサー非依存の処理層。LSM6DSL と LSM6DS3TR-C の両方で同じコードが動作する。

### ファイル構成

| ファイル | 説明 |
|----------|------|
| `imu_types.h` | 型定義 |
| `imu_geometry.c/h` | クォータニオン、ベクトル、行列演算 |
| `imu_stationary.c/h` | 静止判定 |
| `imu_fusion.c/h` | センサーフュージョン（相補フィルタ） |
| `imu_calibration.c/h` | キャリブレーション永続化 |
| `imu_main.c` | NSH コマンド `imu` + デーモン |

---

## 6. センサーフュージョン

pybricks から移植したクォータニオンベースの相補フィルタを使用。

### 初期化

最初の重力ベクトル測定からクォータニオンを生成。

### サンプルごとの処理

1. **キャリブレーション適用**: 加速度のオフセット+スケール補正、ジャイロのバイアス+スケール補正
2. **重力推定**: 回転行列の第 3 行から推定重力ベクトルを抽出
3. **誤差信号**: 推定重力と測定重力の外積を計算
4. **静止度**: ブレンディング係数 (0..1) を計算
5. **融合角速度**: `omega_fused = omega_gyro + correction * fusion_gain`
6. **クォータニオン積分**: 前進オイラー法 + 正規化

### ヘディング

- **1D ヘディング**: 軸ごとの回転積分
- **3D ヘディング**: x 軸の水平面への atan2 射影

---

## 7. キャリブレーション

### ジャイロバイアス

静止状態時の指数平滑法で推定。

### 加速度キャリブレーション

6 姿勢重力測定（各軸の正負方向）でオフセットとスケールを算出。

### ジャイロスケール

軸ごとの補正係数。1 回転がおよそ 360 度になるよう調整。

### 保存形式

設定は `/data/imu_cal.bin` にバイナリファイルとして永続化。

### デフォルト値

| パラメータ | デフォルト値 |
|-----------|-------------|
| 重力 | ±9806.65 mm/s² |
| スケール | 360 |
| ジャイロ閾値 | 2 deg/s |
| 加速度閾値 | 2500 mm/s² |

---

## 8. 静止判定

### アルゴリズム

1. 125 サンプルの遅い移動平均をベースラインとして計算
2. 各サンプルをジャイロ・加速度閾値と比較
3. 約 1 秒間連続して静止と判定された場合:
   - バイアス推定コールバックを呼び出し
   - 実際のサンプル時間を測定

## 10. Hub 移行ガイド

SPIKE Prime Hub への移行時に LSM6DSL ドライバを LSM6DS3TR-C ドライバに差し替え。

### 主な差異

| 項目 | LSM6DSL (B-L4S5I) | LSM6DS3TR-C (Hub) |
|------|---------------------|-------------------|
| I2C アドレス | 0x6A | 0x6A |
| I2C バス | I2C2 (PB10/PB11) | I2C2 (PB10/PB3) |
| WHO_AM_I | 0x6A | 0x6A |
| ODR | 最大 6.66 kHz | 最大 6.66 kHz |

### 軸符号補正

Hub PCB の実装方向に合わせて軸符号を補正する必要あり。

| 軸 | 符号 |
|----|------|
| X | -1 |
| Y | +1 |
| Z | -1 |

### 処理ライブラリ

`apps/imu/` のセンサーフュージョン・キャリブレーションコードは変更不要。uORB トピックインターフェースが同一のため、ドライバのみ差し替え。
