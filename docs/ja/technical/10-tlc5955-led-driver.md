# TLC5955 LED ドライバ プロトコル詳細

## 1. 概要

TLC5955 は TI の 48 チャンネル 16bit PWM LED ドライバ。SPI ライクなシフトレジスタプロトコルで通信。

| 項目 | 値 |
|---|---|
| チャンネル数 | 48 (16 RGB グループ × 3) |
| PWM 分解能 | 16 bit |
| シフトレジスタ長 | 769 bit (97 bytes) |
| 通信 | SPI (SCLK + SIN) + LAT (ラッチ) |
| PWM クロック | GSCLK (外部入力) |

---

## 2. SPIKE Hub 接続

| 信号 | 接続先 | 備考 |
|---|---|---|
| SPI | SPI1 | DMA2 Stream2 (RX, ch3) / Stream3 (TX, ch3) |
| LAT | PA15 (GPIO) | ラッチ信号 |
| SPI モード | CPOL=0, CPHA=0 (Mode 0) | MSB first, 8bit, prescaler /4 |

---

## 3. シフトレジスタフォーマット (769 bit)

### 制御データラッチ

| ビット位置 | フィールド | 説明 |
|---|---|---|
| 768 | ラッチ選択 | `1` = 制御データ |
| 767-760 | マジックバイト | `0x96` (識別用) |
| 370 | LSDVLT | LED 短絡検出電圧 |
| 369 | ESPWM | ES-PWM モード (有効推奨) |
| 368 | RFRESH | 自動リフレッシュ |
| 367 | TMGRST | タイミングリセット |
| 366 | DSPRPT | 自動表示リピート (有効推奨) |
| 365-345 | BC (3×7bit) | 全体輝度制御 (Blue, Green, Red) |
| 344-336 | MC (3×3bit) | 最大電流設定 (Blue, Green, Red) |
| 335-0 | DC (48×7bit) | ドット補正 (チャンネル別) |

### グレースケールデータラッチ

| ビット位置 | フィールド | 説明 |
|---|---|---|
| 768 | ラッチ選択 | `0` = グレースケールデータ |
| 767-0 | GS データ | 48 チャンネル × 16 bit = 768 bit |

チャンネル順序: CH47 (MSB 側) → CH0 (LSB 側)。逆順マッピング。

---

## 4. 初期化シーケンス

pybricks の実装 (`pwm_tlc5955_stm32.c`):

1. 制御データラッチ (97 bytes) を SPI DMA で送信
2. LAT ピン (PA15) をトグル (HIGH → LOW)
3. 制御データラッチを**再送** (最大電流設定を確定するため必要)
4. LAT 再トグル
5. グレースケール更新ループ: 値変更時にグレースケールデータ送信 + LAT トグル

### SPIKE Hub の設定値

```c
最大電流: 3.2 mA (TLC5955_MC_3_2)
ES-PWM: 有効
自動表示リピート: 有効
```

## 5. チャンネル → LED マッピング

### 5×5 LED マトリクス (TLC5955 チャンネル番号)

| | Col 0 | Col 1 | Col 2 | Col 3 | Col 4 |
|---|---|---|---|---|---|
| Row 0 | CH38 | CH36 | CH41 | CH46 | CH33 |
| Row 1 | CH37 | CH28 | CH39 | CH47 | CH21 |
| Row 2 | CH24 | CH29 | CH31 | CH45 | CH23 |
| Row 3 | CH26 | CH27 | CH32 | CH34 | CH22 |
| Row 4 | CH25 | CH40 | CH30 | CH35 | CH9 |

5×5 マトリクスは**単色** (各ピクセル 1 チャンネル)。25 チャンネル使用。

### ステータス LED (RGB)

| LED | R | G | B |
|---|---|---|---|
| Status Top (中央ボタン上) | CH5 | CH4 | CH3 |
| Status Bottom (中央ボタン下) | CH8 | CH7 | CH6 |
| バッテリーインジケータ | CH2 | CH1 | CH0 |
| Bluetooth インジケータ | CH20 | CH19 | CH18 |

4 個の RGB LED × 3 チャンネル = 12 チャンネル。色補正係数: R=1000, G=170, B=200。

### 未使用チャンネル

CH10-17, CH42-44 (計 11 チャンネル未使用)

### 合計

25 (マトリクス) + 12 (ステータス) + 11 (未使用) = 48 チャンネル

## 6. NuttX ドライバ設計方針

### 必要な構成

1. **SPI1 マスタードライバ** + DMA (NuttX 標準ドライバで対応可能)
2. **GPIO 制御** (PA15 = LAT ピン)
3. **カスタム TLC5955 ドライバ** (キャラクタデバイス)

### ドライバインターフェース案

```
/dev/ledmatrix    — 5×5 マトリクス制御
/dev/statusled    — ステータス LED 制御
```

または NuttX の USERLED フレームワークを拡張。

### 実装優先度

初期ブリングアップでは TLC5955 は後回し。まずは GPIO で直接制御可能な LED があるかを確認 (TLC5955 経由でない LED は存在しない可能性が高い)。
